介紹

為了減少跨職位、部門、甚至是外包的溝通，讓大家對畫面的想像一致、溝通無分歧，所以建立屬於天下的一套網站設計系統。定義產品整體的設計，由一組共享的、整合的元素及原則所組成，讓設計跟開發可以模組化，統一 UI 元件及微互動方式一致性。

參與角色

  • 專案統籌、
  • 前端工程、
  • Vue.js

其他參與角色

  • UI 設計師、
  • 平面設計師

我想要解決什麼？

一致性

  • UI 元件及互動⽅式相同
  • 維持產品的連貫性
  • 幫助建立品牌風格

效率性

  • 設計元件庫
  • 隨時使⽤元件、同步更新

跨部門語言

  • 減少來回溝通或落差

互動示範區

第〇版公司主要用於產品部門使用，設計系統可以解決上述想解決的問題，但是這個頁面的互動性、加值服務較少。在第一版公開釋出的版本上，選擇比較多狀態、規則複雜的元件上，如文字框、頁碼等，新增了互動示範區。
